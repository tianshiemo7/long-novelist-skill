## 核心设计理念

本技能专为**百万字级**长篇网文创作设计，区别于短篇或中篇工具，核心差异在于：

| 理念 | 说明 |
|------|------|
| **持久化至上** | 所有状态、角色、伏笔全部落盘为文件，确保跨会话一致性。新会话开始时自动检测写作进度 |
| **用户主导节奏** | 不自动连写。每章写完后展示并等待用户反馈，支持修改迭代后继续 |
| **细纲先行** | 先写整体大纲（剧情走向 + 章节划分），再生成每章细纲（核心事件 + 引子 + 钩子 + 伏笔操作） |
| **起承转合** | 长篇结构必须有明确的起（铺垫）、承（发展）、转（转折）、合（高潮收束），每卷内部同样适用 |
| **伏笔管理系统** | JSON 伏笔清单追踪每个伏笔的铺设与回收状态，分阶段铺设、集中回收，杜绝"挖坑不填" |

## 功能特性

### 交互式配置

通过 13 步递进式问答，你自主控制一切创作参数：

- **题材选择**：玄幻/仙侠、都市/现实、悬疑/惊悚、科幻/末世、历史/架空、言情/纯爱、奇幻/魔幻、游戏/电竞、无限流/综漫
- **主角设定**：姓名、性别、年龄、性格核心、致命缺陷、成长弧线
- **世界观规模**：从单体城市到多元宇宙
- **每章字数**：2000–8000 字自由设定（默认 4000 字）
- **写作风格**：白描/华丽/幽默/冷峻/热血/温情等
- **模仿对象**：支持联网搜索目标作者的作品样本，提取风格卡片并三级模仿（词汇/句式/叙事节奏）

### 大纲系统

```
整体大纲（起承转合四段）
    └── 细纲（每章核心事件 + 章首引子 + 结尾钩子 + 伏笔操作）
```

- 整体大纲按起承转合四阶段分配章节，每阶段占比可控
- 细纲精确到每章：本章发生什么事件、引入/回收哪些伏笔、结尾用什么钩子
- 支持大纲修改——发现前面规划有问题时，可局部调整而不影响已写章节

### 角色管理系统

每个角色拥有独立的档案文件，采用 **YAML frontmatter + Markdown 正文**双层结构：

- **静态信息**：姓名、年龄、外貌、性格特征、致命缺陷、背景故事
- **动态状态**：当前位置、当前目标、情绪状态、身体状况
- **关系网络**：与其他角色的关系（信任/敌对/爱慕/师徒等）及关系演化日志
- **状态变更日志**：每次出场后自动更新，记录角色成长轨迹
- **角色索引**：按 ID/章节/分组快速定位所有角色

检测规则：连续 N 章未出场的角色会被标记提醒，防止"角色消失"。

### 伏笔追踪

JSON 格式的伏笔清单，每条伏笔记录：

```json
{
  "id": "FS001",
  "name": "主角身世之谜",
  "type": "人物背景",
  "status": "铺设中",
  "planted_chapter": 3,
  "hints": ["第3章玉佩异象", "第7章老仆欲言又止"],
  "reveal_chapter": null,
  "priority": "高"
}
```

- **铺设阶段**：分散埋在多个章节中，每次只给一点线索
- **回收阶段**：在高潮/关键转折处集中揭示
- **定期检查**：每写完一卷检查未回收伏笔数量，防止堆积

### 跨会话恢复

每次离开会话前自动更新以下文件，下次启动时无缝恢复：

- `08-写作动态.md` —— 第一恢复入口，包含当前位置、近 5 章摘要、活跃角色列表、伏笔紧急清单
- `05-写作计划.json` —— 确认写作进度，定位下一章
- `00-剧情梗概.md` —— 百万字剧情总览，快速回顾

### 用户偏好学习

自动记录你的写作偏好和反馈模式，跨会话持久化：

- 风格偏好（你多次使用的写作风格）
- 字数偏好（你实际满意的每章字数范围）
- 反馈模式（你最常提出的修改要求类型）
- 错误恢复（记录上次配置冲突，避免重复）

在 Claude Code 中输入以下任一指令：

```
/long-novelist
```

```
"我想写一部长篇中文小说"
"帮我开始写网文"
"我要创作一个修仙故事"
```

首次启动会引导你完成 13 步交互式配置，之后每次会检测是否有未完成的项目并询问是否继续。

## 完整工作流

### 第 0 步：初始化与续写检测

进入 Skill 时自动执行：

1. 扫描 `./long-novelist/` 目录，查找所有小说项目
2. 检查每个项目的 `05-写作计划.json` 中的 `status` 字段
3. 按状态分类展示：

| status | 含义 | 展示示例 |
|--------|------|---------|
| `configuring` | 配置未完成 | "《剑道独尊》：配置中（8/13 步完成）" |
| `planning` | 大纲已生成，待确认 | "《剑道独尊》：大纲已完成，等待确认" |
| `writing` | 写作进行中 | "《剑道独尊》：已写至第 42 章，共 168,000 字" |
| `completed` | 已完结 | "《剑道独尊》：已完结（120 章 / 480,000 字）" |

4. 如有进行中的项目，询问：继续写作 or 开新项目？

### 第一阶段：自主配置（13 步交互问答）

通过四层递进式问答收集所有创作参数：

**第一层：核心定位（Q1–Q3）**
- Q1：选择题材（9 种大类可选）
- Q2：主角设定（姓名、性别、性格核心、致命缺陷）
- Q3：世界观规模（单体城市 → 多元宇宙）

**第二层：写作规格（Q4–Q7）**
- Q4：每章字数（2000 / 3000 / 4000 / 5000+）
- Q5：总字数目标（30 万 / 50 万 / 100 万 / 自定义）
- Q6：更新频率预期（日更 / 隔日更 / 周更 / 不定期）
- Q7：写作风格偏好（白描 / 华丽 / 幽默 / 冷峻 / 热血 / 温情）

**第三层：风格定制（Q8–Q10）**
- Q8：是否有模仿对象？（有 → 进入联网搜索分析流程 / 无 → 跳过）
- Q9：语言规范（偏文言 / 现代白话 / 轻松口语 / 精炼文学）
- Q10：叙事视角（第一人称 / 第三人称限知 / 第三人称全知 / 多视角切换）

**第四层：高级设定（Q11–Q13）**
- Q11：感情线比重（无 / 辅助 / 并重 / 主线）
- Q12：爽点密度（高频 / 中等 / 低频）
- Q13：分卷偏好（按字数分卷 / 按剧情分卷 / 不设分卷）

### 第二阶段：大纲与细纲规划

用户提供创意种子 → AI 加工完善 → 用户确认 → 生成全部项目文件：

1. **创意输入**：你只需要一句话或几个关键词（如"一个被宗门追杀的少年获得了一本逆天功法"）
2. **AI 加工**：展开为完整的起承转合大纲，合理分配章节
3. **用户确认**：查看大纲、提出修改意见、调整剧情走向
4. **细纲生成**：确认后生成每章细纲，包含：
   - 本章核心事件
   - 章首引子（吸引读者进入本章）
   - 结尾钩子（让读者迫不及待看下一章）
   - 伏笔操作（本章铺设哪些伏笔、回收哪些伏笔）
5. **文件创建**：一次性创建所有项目文件（角色档案 / 伏笔清单 / 写作计划等）

### 第三阶段：逐章写作

用户驱动节奏，AI 只在你明确指令时才写下一章：

1. **写前准备**：自动读取角色档案、细纲、伏笔清单、风格设定等上下文
2. **开始写作**：你说"开始写作"或"写下一章"，AI 开始创作
3. **章节输出**：TXT 格式纯文本输出到 `chapters/` 目录，不含 Markdown 标记
4. **字数检查**：自动统计中文字数，确保在设定范围内
5. **状态更新**：自动更新角色信息、伏笔状态、写作动态
6. **等待反馈**：你可以要求修改、重写，或直接说"继续"
7. **钩子预告**：每章结尾展示下一章的钩子，维持创作动力

### 第四阶段：大纲辅助（按需触发）

当你在写作过程中遇到卡文，可以询问后续大纲建议：

- AI 会联网搜索同题材小说的常见处理方法
- 提供 2–3 个可行的剧情发展方向
- 你选择方向后，更新后续细纲
- 同时处理大纲冲突——如果你有了新想法，AI 帮助调整受影响的后文章节

```
│   ├── 角色索引.md              # 按 ID/章节/分组快速查找

│   └── 角色/
│       └── {角色名}/
│           └── 角色信息.md       # YAML frontmatter + Markdown（静态+动态信息）
├── 04-伏笔清单.json            # 所有伏笔及回收状态追踪

├── chapters/                   # 章节内容（TXT 格式）
│   ├── 第01章-xxx.txt
│   ├── 第02章-xxx.txt
│   └── ...
└── summaries/                  # 已完成内容的分卷摘要
    ├── 卷一摘要.md
    ├── 卷二摘要.md
    └── ...
```

## 五大黄金法则

### 一、展示而非讲述

用动作、对话、细节来表现，不要直接陈述。

> **坏**："他很愤怒。"
>
> **好**："他捏碎了手中的茶杯，碎片扎进掌心，血顺着指缝往下淌，他浑然不觉。"

### 二、冲突驱动剧情

每章必须有至少一个冲突点（人物冲突 / 内心冲突 / 环境冲突），推动剧情前进。没有冲突的章节应删除或重构。

### 三、悬念承上启下

每章结尾必须有钩子，让读者产生"下一章必须看"的冲动。钩子类型包括：信息悬念、情感悬念、动作悬念、反转悬念等。

### 四、人物驱动情节

情节服务于人物成长。每个人物的行为必须符合其性格核心和致命缺陷。让角色的选择推动故事发展，而不是作者强加安排。

### 五、伏笔即是契约

铺设的每一个伏笔都是对读者的承诺。伏笔必须记录在案，并在适当时候回收。决不允许"挖坑不填"。

## 起承转合：长篇结构铁律

百万字长篇中文小说必须遵循起承转合四阶段结构：

| 阶段 | 占比 | 核心任务 | 写作重点 |
|------|------|---------|---------|
| **起** | ~20% | 建立世界观、引入主角、铺设核心冲突 | 人物弧线起点、日常世界、激励事件 |
| **承** | ~30% | 冲突升级、人物成长、支线展开 | 能力提升、盟友加入、反派施压 |
| **转** | ~30% | 重大转折、真相揭露、人物蜕变 | 背叛/牺牲、世界观颠覆、主角觉醒 |
| **合** | ~20% | 最终对决、伏笔回收、结局收束 | 高潮决战、情感高潮、余韵 |

每卷内部也应包含小型起承转合，形成"大四段套小四段"的节奏结构。
