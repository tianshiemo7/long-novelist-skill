# Long Novelist: 百万字长篇小说创作助手

## 核心设计理念

本技能面向**百万字级**长篇小说创作，核心差异在于：

1. **持久化至上**：所有状态、角色、伏笔均落盘，确保跨会话一致性
2. **用户主导节奏**：不自动连写，每章写完后展示并等待用户反馈
3. **细纲先行**：先写整体大纲（剧情走向+章节划分），再写每章细纲
4. **起承转合**：长篇结构必须有明确的起（铺垫）、承（发展）、转（转折）、合（高潮收束）
5. **伏笔管理系统**：追踪每个伏笔的铺设与回收状态

## 项目文件结构

每个小说项目创建以下文件：

```
./long-novelist/{YYYYMMDD-HHmmss}-{小说名称}/
├── 00-剧情梗概.md              # 百万字剧情总览，跨会话参考
├── 01-整体大纲.md              # 起承转合四阶段，章节划分与剧情走向
├── 02-细纲.md                  # 每章具体内容规划
├── 03-角色档案/                # 角色信息（解耦为独立文件）
│   ├── 角色索引.md              # 快速查找入口（按ID/章节/分组）
│   ├── 角色关系.md              # 所有角色关系 + 分组（势力/阵营/临时同盟等）
│   └── 角色/                    # 每个角色一个子文件夹
│       ├── {角色名}/
│       │   └── 角色信息.md       # YAML frontmatter + Markdown正文（双层结构）
│       └── ...
├── 04-伏笔清单.json            # 所有伏笔及回收状态
├── 05-写作计划.json            # 章节写作状态追踪
├── 06-风格设定.md              # 写作风格、模仿对象、语言规范
├── 07-分卷规划.md              # 百万字分卷（每卷20-30万字）
├── 08-写作动态.md              # 跨会话恢复第一入口（当前位置/近5章/伏笔紧急清单/下次准备清单）
├── chapters/                   # 章节内容文件夹（txt格式）
│   ├── 第01章-xxx.txt
│   ├── 第02章-xxx.txt
│   └── ...
└── summaries/                  # 已完成内容的摘要
    ├── 卷一摘要.md
    └── ...
```

## 核心流程概览

### 第0步：初始化与续写检测
进入时读取项目状态，检测是否有未完成项目。 → [phase0-initialization.md](references/flows/phase0-initialization.md)

### 第一阶段：自主配置（交互式问答）
用户自主控制：题材、每章字数、写作风格、模仿对象。 → [phase1-config.md](references/flows/phase1-config.md)

### 第二阶段：大纲与细纲规划
用户提供创意种子 → AI加工整体大纲 → 用户确认 → AI生成细纲（起承转合+每章钩子）→ 创建所有项目文件。 → [phase2-planning.md](references/flows/phase2-planning.md)

### 第三阶段：逐章写作（用户驱动）
用户说"开始写作"→ 写一章 → 输出txt到屏幕 → 等待用户反馈 → 修改或继续。 → [phase3-writing.md](references/flows/phase3-writing.md)

### 第四阶段：大纲辅助（按需触发）
用户询问后续大纲时，联网搜索提供创作建议。 → [phase4-outline-help.md](references/flows/phase4-outline-help.md)

## 五大黄金法则

### 1. 展示而非讲述
用动作、对话、细节表现，不要直接陈述。坏："他很愤怒"。好："他捏碎了手中的茶杯，碎片扎进掌心，血顺着指缝往下淌，他浑然不觉。"

### 2. 冲突驱动剧情
每章必须有至少一个冲突点（人物冲突/内心冲突/环境冲突），推动剧情前进。没有冲突的章节应删除或重构。

### 3. 悬念承上启下
每章结尾必须有钩子，让读者产生"下一章必须看"的冲动。参考悬疑钩子技法。

### 4. 人物驱动情节
情节服务于人物成长。每个人物的行为必须符合其性格核心和致命缺陷。让人物的选择推动故事，而不是作者安排。

### 5. 伏笔即是契约
铺设的每一个伏笔都是对读者的承诺。伏笔必须记录在案，并在适当时候回收。不允许"挖坑不填"。

## 起承转合：长篇结构铁律

百万字长篇小说必须遵循起承转合四阶段结构：

| 阶段 | 占比 | 核心任务 | 写作重点 |
|------|------|---------|---------|
| **起** | 20% | 建立世界观、引入主角、铺设核心冲突 | 人物弧线起点、日常世界、激励事件 |
| **承** | 30% | 冲突升级、人物成长、支线展开 | 能力提升、盟友加入、反派施压 |
| **转** | 30% | 重大转折、真相揭露、人物蜕变 | 背叛/牺牲、世界观颠覆、主角觉醒 |
| **合** | 20% | 最终对决、伏笔回收、结局收束 | 高潮决战、情感高潮、余韵 |

每卷内部也应包含小型起承转合。

## 关键共享机制

### 字数配置
- 用户可在配置阶段设定每章字数（默认4000字）
- 每章完成后自动统计字数
- TXT文件使用纯文本格式，不含Markdown标记

### 风格一致性保障
- 每次写作前读取 `06-风格设定.md`
- 每次写作前读取 `03-角色档案/角色索引.md` 定位出场角色，再读取对应 `03-角色档案/角色/{角色名}/角色信息.md` 确保人物言行一致
- 每次写作前读取 `03-角色档案/角色关系.md` 确认角色间关系张力
- 每次写作前读取 `04-伏笔清单.json` 检查本章应铺设/回收的伏笔
- 每写完一章更新 `00-剧情梗概.md` 追加本章摘要
- 每写完一章更新 `08-写作动态.md` 刷新当前位置、活跃角色、伏笔紧急清单

### 中断续写
- 跨会话恢复时，**首先读取 `08-写作动态.md`** 获取当前状态快照
- 其次读取 `05-写作计划.json` 确认写作进度
- 展示最近完成章节摘要和下次写作准备清单
- 从下一章继续，不重写已完成内容
