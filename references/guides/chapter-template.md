# 章节创作模板

## TXT 文件输出格式

每章写入 `chapters/第{XX}章-{章节标题}.txt`，格式如下：

```
═══════════════════════════════════
《{小说名称}》
第{X}卷：{卷名}
第{XX}章 {章节标题}
═══════════════════════════════════

【章首引子】
（50-150字引子文字）

（正文内容，目标字数为用户设定值）

（章末钩子）

═══════════════════════════════════
（可选）作者注：本章伏笔/关键信息
═══════════════════════════════════
```

---

## 章节内部结构

### 1. 章首引子（50-150字）
从引子七式中选择（悬念对话/闪前碎片/倒计时/神秘独白/反差场景/未完成动作/意象伏笔）。

> 作用：读者还没开始读正文就勾起好奇心。

### 2. 正文开头（前10-15%）
使用十种开头技巧之一。第一段必须建立冲突或悬念。

### 3. 发展推进（中间60-70%）
本章核心内容，必须推进主线或深化人物。

### 4. 高潮时刻（后10-15%）
本章情感或动作最高点。

### 5. 结尾钩子（最后5-10%）
从悬念钩子十三式中选择。不可与本章开头使用同一个钩子类型。

---

## 每章必须包含的元素

- [ ] 章首引子
- [ ] 至少2个张力波峰
- [ ] 至少1个读者预期外的转折或信息
- [ ] 对话占比不低于30%
- [ ] 章末悬念钩子
- [ ] 至少1处与前后文的呼应（呼应旧伏笔或埋新伏笔）

---

## 单章质量自查三问

1. 删掉这章，故事是否漏掉关键信息？（是 → 保留；否 → 重写）
2. 读完这章最后一句话，读者是否必须翻下一页？（否 → 重写钩子）
3. 这章中的人物行为是否符合角色档案中的设定？（否 → 修改）
