# 伏笔管理指南

## 伏笔的三个原则

### 1. 有铺必有收
铺设的每一个伏笔都是对读者的契约。不回收的伏笔是对读者的欺骗。

### 2. 分散铺设，集中回收
- 铺设分散在全书中（不要太集中）
- 回收分批次（转阶段集中回收一批，合阶段回收余下）
- 一批伏笔回收完后，要有新的悬念顶上

### 3. 伏笔分层
| 层级 | 读者能注意到吗 | 示例 |
|------|--------------|------|
| 明伏笔 | 多数读者注意到 | "我还会回来的" — 读者知道这人会再出现 |
| 暗伏笔 | 部分读者注意到 | 第三章路人说的一句话，第十章的真相 |
| 隐藏伏笔 | 回看才能发现 | 一个奇怪的眼神、一件不搭的物品 |

## 伏笔铺设技巧

### 技巧1：日常化植入
在最普通的场景中植入伏笔，让读者毫无警觉。

> 坏：角色A说"XX其实隐藏着一个大秘密" — 太明显
> 好：角色A在泡茶时，无意提起"这茶叶是XX送的" — 后面揭露XX在茶里下毒

### 技巧2：多重信息
一句话同时传递表面信息和伏笔信息。

> "你父亲是个好人" — 表面是赞扬
> 后面揭露：其实说话者知道父亲做了什么，"好人"是反讽

### 技巧3：物品载体
伏笔不一定是信息，可以是一个物品的出现和再次出现。

> 第2章：主角在地摊上买了一块玉佩，摊主说"不值钱，戴着玩"
> 第50章：玉佩在关键时刻发出光芒，暴露了其真实身份

### 技巧4：反常细节
一个反常但容易被忽略的细节。

> 第8章：反派在所有人愤怒时，嘴角微微上扬（被忽略了）
> 第80章：揭露这一切都是反派的局

## 伏笔回收技巧

### 技巧1：延迟揭示
伏笔铺设和回收之间的章节越多，"原来如此"的效果越强。但要让读者在回看时能对应上。

### 技巧2：层层揭示
一个伏笔分多次回收，每次揭开一层真相。

> 铺设："你师父不是普通人"
> 第一次回收（第30章）：师父曾是某宗门的长老
> 第二次回收（第60章）：师父离开宗门的原因与主角有关
> 第三次回收（第90章）：师父是主角的亲生父亲

### 技巧3：代价揭示
伏笔的真相必须付出代价才能知道。不要让主角轻松发现真相。

> 坏：角色B直接告诉主角"其实你是XX之子"
> 好：主角在濒死状态下看到了记忆片段，自己拼出了真相

### 技巧4：反转式回收
读者以为自己猜到了伏笔的含义，但真正的含义是另一个。

> 预设：读者以为神秘人A是反派卧底
> 回收：神秘人A确实是卧底，但他卧底的对象是反派，他是主角阵营的人

## JSON管理操作

### 添加新伏笔
写作时想到新伏笔，立即添加到 `04-伏笔清单.json`：

```json
{
  "id": "fs_XXX",
  "description": "...",
  "plantedChapter": 当前章号,
  "plantedScene": "具体场景",
  "type": "...",
  "importance": "...",
  "status": "pending",
  "resolvedChapter": null,
  "resolvedScene": null,
  "revealPlan": "大致计划什么时候回收",
  "notes": ""
}
```

### 标记伏笔已回收
写完后更新：

```json
{
  "status": "resolved",
  "resolvedChapter": 当前章号,
  "resolvedScene": "回收场景"
}
```

### 定期检查
每写完20章，扫描 `04-伏笔清单.json`：
- 检查是否有超过50章未回收的伏笔（提醒用户）
- 确认回收计划仍然合理
- 标记已废弃的伏笔（status: "abandoned"）
