# 第四阶段：大纲辅助（按需触发）

当用户在写作过程中询问"后续大纲怎么编"、"下一卷怎么发展"、"卡文了怎么办"等问题时触发。

---

## 触发检测

识别以下用户意图：
- "后续大纲怎么写" / "下面怎么发展"
- "卡文了" / "没灵感了" / "不知道怎么往下写"
- "帮我想想下一卷的剧情"
- "这个转折怎么设计"
- "XX角色的后续走向有什么建议"
- "帮我想几个情节桥段"

---

## 联网搜索辅助

当用户询问创作建议时，使用 `WebSearch` 工具搜索相关资源：

### 搜索策略

| 用户问题类型 | 搜索关键词模板 |
|------------|--------------|
| 情节卡壳 | "{题材} 小说剧情桥段设计" "{题材} 经典情节套路" |
| 人物走向 | "小说人物弧线设计" "{角色类型} 角色成长路线" |
| 转折设计 | "小说反转情节设计技巧" "起承转合 转折写法" |
| 高潮设计 | "{题材} 小说高潮写法" "最终决战 情节设计" |
| 世界观扩展 | "{题材} 世界观设定" "{题材} 修炼体系设计" |
| 模仿学习 | "{模仿对象} 剧情结构分析" "{模仿对象} 情节设计特点" |

### 搜索后处理

1. 提取搜索结果中的具体技法、桥段模板、案例
2. 结合用户小说的具体设定，给出2-3个**定制化**的建议方案
3. 不要直接照搬搜索结果，必须根据用户已有剧情进行适配
4. 每个建议方案标注出处（"参考自XXX"）

---

## 大纲辅助输出格式

```
🔍 关于 [用户问题]，我搜索了一下，给你几个思路：

**方案一：[方案名称]**
- 核心设计：[2-3句描述]
- 如何与现剧情衔接：[具体说明]
- 对人物的影响：[具体说明]
- 可能的风险：[需要注意的点]
- 参考来源：[来源说明]

**方案二：[方案名称]**
（同上结构）

**方案三：[方案名称]**
（同上结构）

你觉得哪个方向更有感觉？也可以组合使用。
```

用户选择方向后，将该方向纳入 `02-细纲.md`，更新后续章节的细纲规划。

## 卡文专项处理

当用户明确说"卡文了"，执行以下步骤：

### 1. 诊断卡文原因

分两步诊断（AskUserQuestion 每问最多4个选项）：

**第一问 — 定位问题层面**：
```
Question: 卡在哪个层面？
Header: 卡文诊断(1/2)
Options:
- 不知道下一章写什么（剧情方向卡住）
- 知道方向但写不出来（表达卡住）
- 剧情走偏了不知道怎么绕回来
- 角色不知道怎么发展了
```

**第二问 — 确认具体感受**（如第一问未覆盖完整）：
```
Question: 具体感受是？
Header: 卡文诊断(2/2)
Options:
- 写出来的东西不好看/没感觉
- 节奏不对，太快或太慢
- 对话写不自然/情节衔接生硬
- 其他（请在"Other"中说明）
```

### 2. 针对性建议

根据诊断结果给出建议，同时联网搜索相关解决技巧。

常见卡文原因的处理：

| 卡文原因 | 处理方式 |
|---------|---------|
| 不知道下一章写什么 | 回到细纲；如果细纲也卡住了，用搜索找类似情节桥段 |
| 写不出来 | 降低标准，先写出"差"的版本；先写对话骨架再补充描写 |
| 剧情走偏 | 回顾整体大纲，判断是"回的来"还是"需要改大纲" |
| 角色不知道发展 | 回到角色档案，用角色的致命缺陷驱动选择 |
| 写出来不好看 | 检查是否有冲突、是否有悬念、对话是否有潜台词 |

## 大纲更新流程

每次大纲辅助结束后，如有修改：
1. 更新 `02-细纲.md` 中受影响的章节
2. 如果整体方向改变，更新 `01-整体大纲.md`
3. 更新 `05-写作计划.json` 的 `updatedAt`
4. 在 `00-剧情梗概.md` 中记录修改日志
