# 第三阶段：逐章写作（用户驱动）

**核心原则：用户主导节奏。每章写完后展示、等待反馈、修改或继续。不自动连写。**

---

## 触发方式

用户输入以下任一关键词触发写作：
- "开始写作"
- "写下一章"
- "继续写"
- "写第X章"

---

## 写前准备（每章必做）

### 1. 读取上下文

```
必须读取以下文件的最新内容：
- 08-写作动态.md → 快速了解当前位置和活跃角色（跨会话恢复第一入口）
- 05-写作计划.json → 确认当前应写哪一章
- 02-细纲.md → 读取本章的细纲规划
- 03-角色档案/角色索引.md → 定位本章出场角色
- 03-角色档案/角色/{角色名}/角色信息.md → 读取出场角色的性格/状态/位置（按索引逐个读取）
- 03-角色档案/角色关系.md → 读取角色间关系张力
- 04-伏笔清单.json → 检查本章应铺设/回收的伏笔
- 06-风格设定.md → 确认语言风格规范
- 00-剧情梗概.md → 读取最近3章摘要，确保连贯
```

**细纲缺失处理**：如果 `02-细纲.md` 中没有当前章节的规划（如写完了已生成的细纲范围），则：
- 暂停写作
- 先为下一批章节（至少10章）生成细纲
- 细纲生成规则参见 [phase2-planning.md 步骤4](phase2-planning.md#步骤4生成细纲)
- 展示新生成的细纲摘要，经用户确认后继续写作

### 2. 更新状态

更新 `05-写作计划.json`：
- 本章 `status` → `"writing"`
- `updatedAt` → 当前时间

## 写作执行

### 章节文件格式

文件名：`chapters/第{XX}章-{章节标题}.txt`

文件内容结构：

```
第XX章 {章节标题}

【章首引子】
（50-150字引子文字）

（正文内容，目标字数为用户设定值）

（章末钩子）
```

### 写作要求

写作前务必阅读以下指南：
- **章节写作规范**：[chapter-guide.md](../guides/chapter-guide.md) — 十种开头技巧、打破读者预期、中文文学技法
- **对话写作规范**：[dialogue-writing.md](../guides/dialogue-writing.md) — 潜台词、节奏控制、权力博弈
- **内容扩充技巧**：[content-expansion.md](../guides/content-expansion.md) — 字数不足时的扩充方法

1. **严格遵循细纲**：核心事件、出场人物、场景列表必须与细纲一致
2. **人物言行一致**：每个角色的对话和行为必须符合其性格档案
3. **伏笔操作**：按细纲要求铺设/回收伏笔
4. **章首引子**：从引子七式中选择，50-150字（参考 hook-techniques「章首引子七式」）
5. **正文开头**：使用[十种开头技巧](../guides/chapter-guide.md#十种开头技巧)之一，第一段必须有冲突或悬念
6. **对话占比**：不低于30%，每段对话有潜台词或推进情节（参考[对话写作规范](../guides/dialogue-writing.md)）
7. **张力波峰**：全章至少2个，连续500字无冲突必须引入新张力
8. **意外转折**：至少1个读者预期之外的信息或事件（参考[打破读者预期](../guides/chapter-guide.md#打破读者预期长篇)）
9. **章末钩子**：必须使用悬念钩子技法，让读者想看下一章
10. **去AI味**（参考[章节质量检查清单](../guides/chapter-guide.md#章节质量检查清单)）：
    - 删除"璀璨""瑰丽""心潮澎湃"等AI高频词
    - 把抽象情感改为具体动作/对话
    - 长短句交替，避免四字格律堆砌
    - 对话口语化，各有口头禅和说话节奏
    - 用具体细节代替笼统描述
    - 不要总结性语句（"他明白了..."），用行为和对话表达
11. **字数不足时**：使用[内容扩充技巧](../guides/content-expansion.md)自然扩充，严禁注水

### 字数管理

- 写完后统计中文字符数（不计标点空格）
- 允许±15%的浮动

## 写作后：展示与反馈循环

### 步骤A：输出章节到屏幕

写完一章后，将完整章节内容直接显示在对话中。

格式：

```
═══════════════════════════════════
📖 《小说名称》第X章：章节标题
═══════════════════════════════════

[章节完整内容]

═══════════════════════════════════
📊 本章统计
- 字数：XXXX字
- 伏笔铺设：X个（[简要说明]）
- 伏笔回收：X个（[简要说明]）
- 钩子类型：[钩子类型]
═══════════════════════════════════
```

### 步骤B：询问反馈

展示完章节后，**必须**使用 `AskUserQuestion`：

```
Question: 对本章有什么修改意见吗？
Header: 修改意见
Options:
- 没有意见，继续写下一章
- 有修改意见（请在"Other"中说明）
- 我需要再看一遍，先不决定
```

### 步骤C：处理反馈

**如果用户选择"没有意见"**：
1. 更新 `05-写作计划.json`：本章 `status` → `"completed"`，填入 `wordCount`，更新 `updatedAt`
2. 更新 `00-剧情梗概.md`：追加本章摘要（200字内）
3. **更新角色档案**（强制步骤）：
   - 更新出场角色的 `03-角色档案/角色/{角色名}/角色信息.md`：`lastAppearedChapter`、`status`、`currentLocation`、追加 `stateHistory`
   - 如有关系变化，更新 `03-角色档案/角色关系.md`：追加关系历史、调整张力值、更新最近重大变化
   - 更新 `03-角色档案/角色索引.md`：同步状态和位置变更
   - 每10章执行一次"被遗忘角色"检查
4. 更新 `04-伏笔清单.json`：如有伏笔操作，更新 `status`、`resolvedChapter` 等
5. **更新 `08-写作动态.md`**（强制步骤）：
   - 刷新 `currentChapter`、`currentPhase`
   - 更新"最近完成"表格（保持近5章）
   - 更新"活跃角色"列表
   - 刷新"伏笔紧急清单"（重新计算剩余章数/优先级）
   - 更新"下次写作准备清单"
6. 保存章节txt到 `chapters/` 目录
7. 展示简要确认："第X章已保存。角色状态已更新。写作动态已刷新。"
8. **询问是否写下一章**：

```
Question: 是否继续写下一章？
Header: 继续写作
Options:
- 是，写下一章
- 先不写了，稍后继续
```

如果选择"是"，回到「写前准备」流程，写下一章。

**如果用户选择"有修改意见"**：
1. 仔细阅读用户的修改意见
2. 只修改用户提到的部分
3. 展示修改后的版本并标注修改处
4. 重新询问是否满意

**修改时的规则**：
- 只改用户指出的问题，不要借机修改其他内容
- 如果是大范围修改（超过30%内容需要重写），先让用户确认修改方案
- 修改后保留原文备份（修改前的内容放在chapters目录下命名为 `第XX章-v{版本号}.txt`）
- `feedbackRounds` 计数器+1

## 连续写作模式（可选）

如果用户在配置阶段明确要求"连续写作不中断"，则调整流程：

1. 写完一章 → 简略展示章节概要（200字）+ 钩子 → 不等待反馈，直接写下一章
2. 每5章暂停一次，询问是否继续
3. 用户随时可以说"暂停"切换到逐章确认模式

但在默认模式下，**必须每章写完等待用户反馈**。

## 写作中可用的用户指令

| 用户输入 | 行为 |
|---------|------|
| "开始写作" / "写下一章" | 写当前应写的章节 |
| "写第X章" | 跳到第X章写作（如同跳章，需要先确认） |
| "修改第X章" | 读取第X章并接受修改意见 |
| "重写第X章" | 完全重写第X章 |
| "查看大纲" | 展示当前大纲 |
| "查看第X章" | 展示第X章内容 |
| "添加伏笔" | 在伏笔清单中添加新伏笔 |
| "更新角色" | 修改角色档案 |
| "生成后续细纲" | 触发下一卷细纲生成 |
