# 第0步：初始化与续写检测

**每次触发本SKILL时首先执行：**

## 1. 扫描项目目录

扫描 `./long-novelist/` 目录，查找所有项目文件夹：

```bash
ls -lt ./long-novelist/
```

对于每个项目，检查 `05-写作计划.json` 中的 `status` 字段。

## 2. 项目状态分类

| status 值 | 含义 | 展示 |
|-----------|------|------|
| `configuring` | 配置未完成 | "《XX》：配置中（X步/X步完成）" |
| `planning` | 大纲已生成，待确认 | "《XX》：大纲已完成，等待确认" |
| `writing` | 写作进行中 | "《XX》：已写 X/Y 章，下一章：第Z章" |
| `completed` | 全部完成 | "《XX》：已完成（共Z章）" |

## 3. 展示项目列表

**有未完成项目时**，展示：

> 检测到以下小说项目：
> 
> 1. 《XXX》 — 已写 15/80 章，下一章：第16章 [继续写作]
> 2. 《YYY》 — 大纲已完成，等待确认 [继续配置]
> 
> 你想：A) 继续某个项目  B) 创建新项目

**没有项目时**，直接进入第一阶段配置。

## 4. 续写准备

用户选择继续项目后：
1. **首先读取 `08-写作动态.md`** 获取当前状态快照（跨会话恢复第一入口）
2. 读取 `05-写作计划.json` 获取进度
3. 读取 `03-角色档案/角色索引.md` 获取角色总览
4. 读取 `00-剧情梗概.md` 的后100行（最新剧情状态）
5. 展示摘要：

> 《XXX》进度摘要：
> - 当前阶段：「承」阶段 | 下一章：第16章
> - 活跃角色：张三（藏经阁）、王五（魔宗分坛）
> - 上章回顾（第15章）：张三在藏经阁发现师父遗书，得知自己是魔宗圣子转世
> - 本章钩子：遗书中提到"另一半在你最信任的敌人手里"——这个人是谁？
> - 🔴 紧急伏笔：fs_002 秘境令牌（第20章前需回收，还剩5章）
> 
> 准备好继续写第16章了吗？输入"开始写作"即开始。

## 5. 新建项目

用户选择新建时，进入第一阶段 → [phase1-config.md](phase1-config.md)
