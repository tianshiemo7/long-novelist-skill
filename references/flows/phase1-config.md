# 第一阶段：自主配置（交互式问答）

用户自主控制一切创作参数。分为四层递进问答。

> **关于 AskUserQuestion 工具限制**：每道题最多4个选项。以下模板中部分题目列出的选项超过4个——执行时AI需自行按优先级精选至4个以内，或将题目拆分为多道子题。优先保留与用户已透露偏好最相关的选项。

---

## 第一层：核心定位（必答，Q1-Q3）

### Q1：选择题材

使用 `AskUserQuestion` 询问：

```
Question: 你想写什么题材的小说？
Header: 题材
Options:
- 玄幻/仙侠 — 修炼、宗门、飞升、长生
- 都市/现实 — 现代生活、职场、商战、情感
- 悬疑/惊悚 — 破案、推理、心理恐惧
- 科幻/末世 — 未来科技、外星文明、末日求生
- 历史/架空 — 古代权谋、历史演义、朝堂江湖
- 言情/纯爱 — 爱情主线，不同时代背景
- 奇幻/魔幻 — 西方魔法、异世界、剑与魔法
- 游戏/电竞 — 虚拟现实、竞技对抗
- 无限流/综漫 — 多世界穿越、副本挑战
```

### Q2：主角设定

```
Question: 主角的性别和类型？
Header: 主角类型
Options:
- 男性主角（独角戏）
- 女性主角（独角戏）
- 双主角（男女双线 / 对手戏）
- 群像戏（多线叙事）
- 自由描述（我已想好角色设定）
```

**追问 Q2a — 职业/身份**：
```
Question: 主角的职业或身份是什么？（简短回答即可）
Header: 职业身份
[自由文本输入]
```

**追问 Q2b — 核心性格**：
```
Question: 主角的核心性格是？
Header: 性格
Options:
- 热血正义（积极、勇敢、有担当）
- 冷静智慧（理性、谋略、高智商）
- 温暖治愈（善良、温柔、有同理心）
- 高冷孤傲（冷漠、独立、强大）
- 阴暗腹黑（心机、算计、复仇）
- 成长逆袭（从弱到强、打脸升级）
- 复杂矛盾（有多面性，不贴标签）
- 自由描述
```

**追问 Q2c — 关键配角**（可选跳过）:
```
Question: 有没有已经想好的关键配角？比如对手、盟友、恋人？（可跳过）
Header: 关键配角
[自由文本输入]
```

### Q3：核心冲突与驱动力

```
Question: 故事的核心冲突类型是什么？
Header: 冲突类型
Options:
- 生死存亡（生存危机、逃出生天）
- 查明真相（寻找答案、揭露秘密）
- 爱情阻碍（追求真爱、克服阻碍）
- 复仇雪恨（复仇计划、伸张正义）
- 权力争夺（竞争上位、资源争夺）
- 成长突破（自我突破、实现价值）
- 守护保护（守护重要的人或事）
- 自由描述
```

**追问 Q3a — 驱动力**：
```
Question: 是什么推动主角不断向前？
Header: 驱动力
Options:
- 复仇（为某人/某事而战）
- 爱情（为所爱之人）
- 责任/使命（不得不做）
- 好奇心/求知欲（想知道真相）
- 生存（活下去）
- 野心（想要得到某样东西）
- 自由描述
```

## 第二层：规模与规格（必答，Q4-Q6）

### Q4：字数与章节

```
Question: 每章的目标字数是多少？
Header: 每章字数
Options:
- 2000-3000字（网文快节奏标准）
- 3000-4000字（推荐，兼顾深度与节奏）
- 4000-5000字（传统出版标准）
- 5000-8000字（大章节，信息密度高）
- 自定义（稍后输入）
```

```
Question: 计划写多少章？（可以是大致范围，百万字通常需要200-400章）
Header: 总章数
[自由文本输入]
```

### Q5：世界观/背景设定

```
Question: 故事发生在什么样的世界？
Header: 世界观
Options:
- 现实世界（当代中国或其他现实背景）
- 架空历史（特定朝代或虚构朝代）
- 完全虚构世界（需自建规则体系，如修炼体系/魔法体系）
- 未来/科幻世界（科技水平、社会结构特殊）
- 自由描述（我已有详细设定）
- 随机生成（根据题材智能生成）
```

**追问**（如选择需要展开的选项）：
```
Question: 这个世界有什么独特的规则或设定要素？（简单描述即可）
Header: 设定要素
[自由文本输入，可跳过]
```

### Q6：视角与人称

```
Question: 叙事视角与人称？
Header: 叙事视角
Options:
- 第一人称（"我"）— 代入感强，适合悬疑/情感
- 第三人称有限视角 — 跟随1-3个主角，最常用
- 第三人称全知视角 — 适合宏大世界观
- 多视角切换 — 每章/每段换不同角色的视角
```

### Q7：语言基调

```
Question: 整体的语言风格和基调？
Header: 语言基调
Options:
- 热血爽文 — 快节奏、强冲突、主角强势
- 深沉厚重 — 文学性强、哲思、慢热
- 轻松幽默 — 搞笑日常、吐槽风、段子手
- 黑暗压抑 — 残酷世界观、生存艰难、灰色道德
- 温馨治愈 — 正能量、日常温情、成长治愈
- 冷峻克制 — 极简描写、克制情感、硬汉风
```

## 第三层：深度定制（选答，Q8-Q10）

### Q8：核心主题

```
Question: 这部小说最想表达的核心主题是什么？（这决定了故事的灵魂）
Header: 核心主题
Options:
- 成长与蜕变（主角的内在变化是核心）
- 正义与复仇（善恶对决、伸张正义）
- 爱与牺牲（感情线驱动，为所爱之人付出一切）
- 权力与欲望（野心、争夺、堕落与救赎）
- 生存与希望（绝境中的人性光辉）
- 自由与束缚（对抗命运/体制/偏见）
- 自由描述
- 随机生成
```

### Q9：目标读者

```
Question: 主要写给谁看？
Header: 目标读者
Options:
- 大众读者（网络文学受众，追求爽感和代入感）
- 青少年读者（偏青春、成长、校园题材）
- 成熟读者（偏好深度、文学性、思想性）
- 不确定 / 自由描述
```

### Q10：模仿对象

```
Question: 有没有想模仿/参考的作家或作品？提供名字或作品名即可，我会联网搜索其风格特征。
Header: 模仿对象
[自由文本输入，可跳过]
```

**如果用户提供了模仿对象**，立即使用 `WebSearch` 工具搜索该作家的写作风格特征：

搜索关键词模板：
```
"{作家名} 写作风格 特点"
"{作家名} 写作技巧 语言特色"
"{作品名} 文笔分析 风格点评"
```

**搜索结果整理为风格卡片**，保存到项目 `06-风格设定.md`：
- 句式特点（长短句偏好、节奏感）
- 描写风格（细腻/粗犷、写实/写意）
- 对话风格（口语化程度、个性化程度）
- 叙事节奏（快/慢、信息密度）
- 情感表达方式（直白/含蓄、克制/奔放）
- 代表性段落（如能找到，摘录1-2段范本）

### Q11：特殊要求

```
Question: 还有什么特殊的写作要求吗？（可多选或跳过）
Header: 特殊要求
Multi-select:
- 避免特定套路/桥段 → 稍后详细说明
- 需要特定文化/行业背景 → 稍后详细说明
- 需要加入玄幻修炼体系设计
- 需要加入地图/势力设定
- 需要加入爱情线/友情线
- 需要限制脏话/暴力程度
- 其他 → 稍后详细说明
```

## 第四层：标题生成与确认（Q12-Q13）

### Q12：标题生成

基于用户的所有配置，AI 根据 [title-guide.md](../guides/title-guide.md) 中的五种创作技巧生成3-5个候选标题。

每个候选标题附带：
- 使用的创作技巧名称
- 为何适合这个故事（一句话）

使用 `AskUserQuestion`：
```
Question: 以下是为你的小说生成的候选标题，请选择：
Header: 标题选择
Options:
- 《标题1》 — [技巧名称]，[一句话说明]
- 《标题2》 — [技巧名称]，[一句话说明]
- 《标题3》 — [技巧名称]，[一句话说明]
- 《标题4》 — [技巧名称]，[一句话说明]
- 《标题5》 — [技巧名称]，[一句话说明]
- 重新生成
```

> 用户可通过 "Other" 输入自定义标题。重新生成最多3轮。

### Q13：确认配置

汇总展示所有用户选择和最终标题：

```
═══════════════════════════════════
📖 《小说名称》创作配置总览

题材：[用户选择]
主角：[类型] | [职业/身份] | [性格] | [驱动力]
核心冲突：[冲突类型]
世界观：[世界观设定]
每章字数：[设定] | 总章数：[设定]
叙事视角：[选择] | 语言基调：[选择]
核心主题：[选择] | 目标读者：[选择]
模仿对象：[选择或"无"]
特殊要求：[汇总]

═══════════════════════════════════
```

使用 `AskUserQuestion`：
```
Question: 以上配置是否正确？是否进入大纲规划？
Header: 最终确认
Options:
- 确认，进入大纲规划
- 需要修改（请说明）
```

用户确认后：
1. 标题信息存入对话上下文
2. 保存配置到内存中（后续写入项目文件）
3. 进入第二阶段：大纲与细纲规划 → [phase2-planning.md](phase2-planning.md)
